  1. 1. Go to console.cloud.google.com and sign in with your Google account.
  2. 2. Click the project selector dropdown at the top of the page (it may say "Select a project" or show an existing project name).
  3. 3. Click New Project in the top-right of the dialog.
  4. 4. Name it something like BusyBridge and click Create.
  5. 5. Make sure your new project is selected in the project dropdown before continuing.
  1. 1. In the left sidebar, go to APIs & Services → Library.
  2. 2. Search for Google Calendar API.
  3. 3. Click on it and then click Enable.

This is the only API you need to enable. BusyBridge uses it for reading/writing calendar events and registering webhook notifications.

  1. 1. Go to APIs & Services → OAuth consent screen.
  2. 2.
    Select the user type:
    • Internal — Only users in your Google Workspace can sign in. No app verification needed. Choose this if your client calendars are all within the same Workspace domain.
    • External (recommended) — Users from any Google account can authorize. Required if you need to connect client calendars from outside your Workspace domain (e.g., a client's @theircompany.com calendar). BusyBridge restricts login to your domain at the app level, so this is safe.
  3. 3. Fill in the required fields: App name (e.g., "BusyBridge"), User support email, and Developer contact email.
  4. 4.
    On the Scopes page, click Add or Remove Scopes and add these scopes:
    openid
    Required for authentication
    .../auth/userinfo.email
    Required for authentication
    .../auth/userinfo.profile
    Display name for user accounts
    .../auth/calendar
    Read/write access — used for your main calendar and client calendars
    .../auth/calendar.readonly
    Read-only access — used for personal calendar connections

    You can search for "calendar" to find the calendar scopes. Both calendar and calendar.readonly will appear as "sensitive" scopes — this is expected.

  5. 5.
    If you chose External: on the Test users page, add the email addresses of everyone who will use BusyBridge — both your own workspace users and any client email accounts you plan to connect. You can add up to 100 test users.
  6. 6. Click Save and Continue through the remaining steps.

External apps in "Testing" mode — important limitations:

  • Only users you've added as test users can authorize
  • Refresh tokens expire after 7 days, requiring users to re-authorize
  • To remove these limitations, submit the app for Google verification (takes a few weeks and requires a privacy policy URL)

Domain restriction: Regardless of whether you choose Internal or External, BusyBridge restricts login to users from your Google Workspace domain (set during the next step). External accounts can only be used for connecting client calendars — they cannot log in to the dashboard.

  1. 1. Go to APIs & Services → Credentials.
  2. 2. Click + Create Credentials → OAuth client ID.
  3. 3. Select Web application as the application type.
  4. 4. Name it something like BusyBridge Web Client.
  5. 5.
    Under Authorized redirect URIs, click + Add URI and add each of these URIs:
    {{ public_url }}/setup/step/3/callback
    {{ public_url }}/auth/callback
    {{ public_url }}/auth/connect-client/callback
    {{ public_url }}/auth/connect-personal/callback

    The first URI is used during this setup wizard. The other three are used at runtime for user login, connecting client calendars, and connecting personal calendars.

  6. 6. Click Create. Google will display your Client ID and Client Secret. Copy them into the fields below.
